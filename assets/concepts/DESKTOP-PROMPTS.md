# ProofGoal desktop concept prompts

The reference image for both prompts is `proofgoal-mobile-flow-v1.png`. It defines the visual identity:
warm paper white, black ink, cobalt actions, proof green, red NO state, condensed display headlines,
monospaced receipt details, perforated paper, and barcode/proof motifs.

## Desktop market

Use case: ui-mockup

Asset type: high-fidelity desktop web application screen, 16:9 landscape

Primary request: Extend the ProofGoal mobile prediction-market UI into a practical desktop market
screen. ProofGoal lets a user stake YES or NO on one football stat, receive a receipt, then have the
result verified on Solana by TxLINE. This screen is before the entry cutoff.

Composition: quiet top navigation; one dominant featured market across the left two-thirds; a compact
pool and receipt-preview column on the right. Keep a single-screen hierarchy rather than a sportsbook
dashboard.

Required copy: `PROOFGOAL`, `THE BET THAT SETTLES ITSELF.`, `MARKET`, `ESP + AUT CORNERS > 9`,
`LOCKS IN 12:14`, `YES 62%`, `NO 38%`, `AMOUNT`, `25 USDT`, `EST. PAYOUT 40.32 USDT`, `GET RECEIPT`,
`POOL 2,480 USDT`, `MY RECEIPTS`, and wallet `GD3s...SE8W`.

Constraints: preserve the reference image's typography, palette, paper texture, thick black rules,
receipt perforations, and candid consumer-fintech simplicity. No live betting at 71 minutes, no
order book, no candlesticks, no odds grid, no team logos, no crypto logos, no decorative gradient,
no fake browser chrome, no watermark.

## Desktop proof receipt

Use case: ui-mockup

Asset type: high-fidelity desktop web application screen, 16:9 landscape

Primary request: Extend the ProofGoal mobile UI into the definitive settled-receipt screen. The
receipt is the hero and demonstrates that a football-stat prediction was verified on Solana through
TxLINE without an administrator.

Composition: quiet top navigation; oversized physical thermal receipt on the left; on the right, a
large final-stat progress module, a three-step verification timeline, proof digest, settlement link,
and one dominant payout action. Cryptographic detail is progressively disclosed and never overwhelms
the receipt.

Required copy: `PROOFGOAL`, `RECEIPT #1047`, `YES · OVER 9 CORNERS`, `STAKE 25 USDT`,
`FINAL COUNT 11`, `PROOF VERIFIED`, `PAYOUT +40.32 USDT`, `1 MATCH FINAL`, `2 TxLINE PROOF`,
`3 SOLANA VERIFIED`, `NO ORACLE VOTE · NO ADMIN · NO DISPUTE`, `CLAIM 40.32 USDT`, and
`VIEW ON SOLANA`.

Constraints: preserve the reference image's visual identity exactly. The receipt must look physical,
shareable, and premium. No sportsbook dashboard, no trading charts, no team logos, no crypto logos,
no decorative gradient, no fake browser chrome, no watermark.

## Desktop live locked receipt

Use case: ui-mockup

Asset type: high-fidelity desktop web application screen, 16:9 landscape

Primary request: Show a ProofGoal receipt during a live match after betting has locked. The user is
watching the receipt update, not placing another bet. Make the live count and distance to the target
emotionally immediate while preserving the physical receipt as the hero.

Composition: quiet navigation; oversized locked receipt on the left; huge match clock and corner
progress in the center; a small event tape and locked-pool summary on the right. No betting controls.

Required copy: `PROOFGOAL`, `LIVE · LOCKED`, `RECEIPT #1047`, `YES · OVER 9 CORNERS`,
`ESP 1–0 AUT · 71:24`, `CURRENT COUNT 7`, `NEEDS 3 MORE`, `POOL LOCKED`, `POT 2,480 USDT`,
`EST. PAYOUT 40.32 USDT`, `TxLINE LIVE`, and `ENTRY CLOSED`.

Constraints: preserve the approved ProofGoal visual identity. No YES/NO buttons, no active stake form,
no order book, no charts, no team logos, no crypto logos, no decorative gradient, no browser chrome,
no watermark.

## Desktop receipt archive

Use case: ui-mockup

Asset type: high-fidelity desktop web application screen, 16:9 landscape

Primary request: Show ProofGoal's `MY RECEIPTS` archive. It should feel like a personal drawer of
physical prediction receipts rather than a financial portfolio dashboard. Show a small set of large,
tactile receipt cards in different lifecycle states.

Composition: quiet navigation; oversized `MY RECEIPTS` heading and simple filters; four large receipt
cards arranged with editorial spacing. One live/locked, one verified winner, one verified loss, and
one refundable receipt. A narrow summary strip may show total staked and total claimed.

Required copy: `PROOFGOAL`, `MY RECEIPTS`, `ALL`, `LIVE`, `VERIFIED`, `REFUNDS`, `4 RECEIPTS`,
`TOTAL STAKED 85 USDT`, `TOTAL CLAIMED 40.32 USDT`, `LIVE · LOCKED`, `PROOF VERIFIED`,
`RESULT: NO`, `REFUNDABLE`, `OPEN RECEIPT`, and receipt numbers `#1047`, `#1039`, `#1028`, `#1014`.

Constraints: preserve the approved ProofGoal visual identity and physical thermal-paper treatment.
Keep only four substantial receipts; no tiny data table, no P&L chart, no trading dashboard, no team
logos, no crypto logos, no decorative gradient, no browser chrome, no watermark.
